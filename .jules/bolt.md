## 2024-10-18 - Short-Circuiting Array Filtering for Performance
**Learning:** In large datasets (like `allColors` in `ColorDashboardClient.tsx`), evaluating all conditions in an array filter and returning their logical AND (`return condA && condB && condC`) can cause severe performance penalties if some conditions are expensive (e.g., `toLowerCase().includes()`) and are evaluated unnecessarily when earlier, simpler conditions (like string equality or Set `.has()`) would have already falsified the total expression. Although JS short-circuits `&&` naturally, separating the fast checks with early returns (`if (!fastCheck) return false;`) ensures clarity, prevents complex block allocations, and explicitly guards the expensive string operations.
**Action:** When filtering large arrays, explicitly sequence conditions: evaluate fast exact-match checks and Set lookups first, and return early. Save expensive string manipulations (`toLowerCase`, regex matching) for the end of the filter condition to minimize their execution frequency.
