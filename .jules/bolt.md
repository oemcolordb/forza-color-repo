## 2026-07-02 - [Short-Circuiting Expensive Lookups Against Large Arrays]
**Learning:** In a codebase manipulating a large dataset of items (e.g. 40k+ `allColors`), performing string manipulations like `.toLowerCase().includes()` or scanning through large arrays for every item without order of operations consideration creates noticeable main thread blocks. Applying early returns (short-circuiting) with simpler comparisons (`===`) and O(1) Set lookups beforehand significantly avoids CPU-intensive execution paths.
**Action:** When working with large datasets, always place the cheapest/O(1) condition checks (like boolean flags, strict equality, and Set `.has()` checks) at the top of filter functions to aggressively short-circuit before falling back to O(N) searches or expensive string operations. Similarly, limit the number of items being processed (e.g., via `.slice()`) *before* invoking O(N) operations in a `.map()` callback.
